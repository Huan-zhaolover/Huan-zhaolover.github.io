汇编02 代码执行效率分析

  • if-else / switch 谁的效率高
  • 汇编代码的条数；执行多少条机器指令
  • 代码地址 是连续的，
  • switch 先计算 再跳转 ，7条指令 已空间换时间；消耗更多的内存，节省时间；
    多开辟内存进行计算；
  • linux unix 汇编格式是AT&T格式
  • Windows 汇编是Intel格式
  • ARM64 是CPU架构 有对应的ARM 汇编

汇编03

switch如何计算 7条指令

  • 少量条件 没有优化，还是一样的效率；多条件 进行了优化
  • 跨度小，判断的省略值多
  • 如果 switch 中值 很大，超出了枚举值范围，会对区间判断 （5-1）月 （8 -1）比较
    • (M - A ) 和 (B - A) 比较起来看范围，不在范围内 直接default
    • cmp dword ptr [ebp-0DCh],5 进行比较
    • ja $LN7 + OFh (0D44529h) 如果大于，就进行跳转到default内存地址
  • 跨度比较大
    • 有一块地址 存放需要判断的值（M-A）+ 一块内存地址
    • movzx eax, byte ptr [edx + 1359522Ch]; 将里面的值取出来存放到eax中，执行下一步；byte 一个字节；若edx 是个8 ，取出04，相当于 movzx eax,04
      • 从 0x1359522c开始 存放了所有的枚举项 各对应的 内存存放偏移；
  • jmp dword ptr [eax*4 + 1396218h]; 根据便宜的值计算真是存放枚举项代码的内存地址

  • 0x1396218h 开始 是枚举项，若case只有5个，就存放5块，不是上面的多有的值都开放内存，用一个小子节，存放偏移量/偏移量

  • 如果跨度 很大，枚举值也很大，switch 10000，case1，case8，case 10000；

  • 这样 和if是一样的 没有进行优化

  • a++ 先使用，再加一
  • ++a,先加一 再使用

汇编05. 构造函数 构造器

  • 错误的 默认情况下，编译器会为每一个类生成空的无参数的构造函数
  • 只有在某些特定的情况下，编译器才会成为类生成空的无参数的构造函数，
    • 成员变量在声明的同时进行了初始化
    • 有定义的虚函数
    • 虚继承了其他类
    • 包含了对象类型的成员，且这个成员有构造函数
    • 父类有构造函数
    • 需要做一些额外操作的时候，比比如内存，函数调用，编译器一般
    • 成员变量在声明的同时进行了初始化
    • 包含了对象类型的成员，且这个成员有构造函数
    • 父类有构造函数
